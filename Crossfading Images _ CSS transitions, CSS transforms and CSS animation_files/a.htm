                                              Desolo SubHumus clarise 5 months ago

                                              Actually, none of these need jQuery or JS to work in modern browsers (ie excepted, though I never consider it to actually be modern, regardless of version). The jQuery/JS solutions are for making old versions of browsers render them properly. Essentially, it's purely an ie problem, as it's the only 1 of the major 5 browsers that doesn't automatically update itself. The other 4 are 'evergreen browsers' (self updating with nightly builds).

                                              As far as offset height of nested divs and such, this is only a real problem if you are still stuck on the old dinosaur of pixel perfect design. Honestly, websites don't NEED to look exactly the same in all browsers. They need to function, yes, but if the user chose a certain browser because they like the way it looks and feels, let their browser of choice render it the way they like it. Responsive and progressive are the way to go.

                                              For example, ie doesn't support rounded corners on elements. If you display a picture and use border-radius: 100% on it, but do not include a fallback, ie will display it with the blocky look that many ie users prefer. Everyone else will see the round image, as they are used to seeing in their browser of choice.
